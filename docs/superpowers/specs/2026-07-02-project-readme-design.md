# 專案 README 設計

## 目標

在專案根目錄新增繁體中文 `README.md`，讓第一次接觸 Nginx 的學習者，以及已有基礎、想按五日計畫實作的讀者，都能快速理解這個專案的用途並開始使用。

## 文件策略

採用「快速開始＋五日導覽」結構。根 README 負責說明入口、操作方式與整體路線；詳細知識與實驗步驟仍保留在 `days/`，避免複製每日教材。

## 內容結構

README 依序包含：

1. 專案簡介、學習目標與適合對象。
2. 目前教材完成狀態，明確標示 Day 1 已有完整內容、Day 2–5 目前主要為課綱。
3. 使用前需求，包括 Git、Docker、curl 與 OpenSSL。
4. 從 clone、環境檢查到進入 Day 1 的快速開始方式。
5. 固定學習循環：Prediction、執行、觀察、Fault Injection、記錄與驗收。
6. 五日課程導覽與各日教材連結。
7. 主要目錄結構與重要文件入口。
8. 常用驗證指令與操作注意事項。

## 正確性原則

- 所有連結使用相對路徑，確保 GitHub 與本機閱讀都可用。
- 不宣稱 Day 2–5 已有尚不存在的實驗檔案。
- Docker 僅描述為重現 Nginx Lab 的工具，不將 Docker 納入課程主題。
- 常用指令必須能從現有 Day 1 教材找到依據，或屬於課前環境檢查。
- 不加入尚未存在的一鍵啟動腳本、套件安裝流程或自動化測試指令。

## 驗證方式

- 檢查 README 中所有相對連結指向現有檔案或目錄。
- 對照 Day 1 實驗，確認範例指令、連接埠與設定檔路徑沒有被錯誤概括。
- 搜尋占位文字與失效引用，確認沒有 `TODO`、`TBD` 或不存在的路徑。
- 檢查 Git diff，確保只加入本次文件與必要規格。

## 範圍外

- 不新增或修改 Nginx Lab。
- 不補齊 Day 2–5 教材。
- 不新增 Docker Compose、Makefile 或執行腳本。
- 不重寫既有五日學習方案與每日 README。
