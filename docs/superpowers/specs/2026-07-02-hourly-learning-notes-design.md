# 每小時學習筆記更新規格

## 目的

Day 2 學習過程中，每個 Hour 通過檢核後，立即把成果寫入當日學習筆記，讓教材反映實際進度與心智模型修正。

## 檔案結構

- 學習筆記統一寫入 `days/day-2/notes.md`。
- `days/day-2/README.md` 提供學習筆記入口。
- 各 Hour 依完成順序追加，尚未完成的 Hour 不預先標記完成。

## 每小時記錄內容

每個完成的 Hour 至少包含：

1. 學習目標與核心原理。
2. 實際題目或實驗揭露的易錯點。
3. 修正後可重用的心智模型。
4. 明確的完成狀態。

筆記採濃縮摘要，不保存完整對話逐字稿。必要的 Config、URI Transformation、Prediction 與 Actual Result 則保留為可重現範例。

## 更新時機

只有在該 Hour 的概念題、預測題或實驗達到該時段驗收條件後才更新。更新完成後，再進入下一個 Hour。

## 成功條件

- 學習者可只讀 `notes.md` 重建該 Hour 的核心心智模型。
- 筆記包含學習中實際出現的重要誤判及正確原因。
- README 能直接連到筆記。
- 每個已完成 Hour 都有清楚的「完成」狀態。
